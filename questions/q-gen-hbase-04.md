According to the lecture, which RDBMS features must you be willing to live without when moving to HBase?

- a) Typed columns
- b) Secondary indexes
- c) Row-level storage of data
- d) Transactions and advanced query languages
- e) Storage on disk

### Explanation

The lecture lists typed columns, secondary indexes, transactions and advanced query languages among the RDBMS extras you give up with HBase (a, b, d). HBase still stores data (on HDFS) at the cell level, so c and e are not features you lose.
