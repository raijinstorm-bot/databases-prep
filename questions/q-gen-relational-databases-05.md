A primary key is best described as:

- a) Any set of columns whose values are unique for the table
- b) A minimal set of columns whose values are unique for the whole table
- c) The first column listed in a table
- d) A column that is allowed to contain NULL values

### Explanation

The presentation explains that because duplicate records are not allowed, there must exist a set of columns with unique values across the table, and such a minimal set is named the primary key. It need not be the first column, and primary key columns cannot be left empty (NULL).
