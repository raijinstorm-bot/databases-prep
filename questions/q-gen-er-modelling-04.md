Which characteristics describe a surrogate primary key?

- a) It uses integer values generated from a sequence (1, 2, ...)
- b) It relies on values not used in the real world, often not shown to end users
- c) It is always a composite key
- d) It is a solution when no natural primary key exists

### Explanation

The presentation describes surrogate primary keys as integer values generated from a sequence, relying on values not used in the real world (often hidden from users), and a solution when no natural PK exists or when the natural PK would be composite. A surrogate key is itself a single column, not a composite key, so c is wrong.
