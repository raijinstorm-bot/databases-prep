A user profile in Oracle can be created and assigned to users in order to:

- a) Set limits on the use of database resources
- b) Define log-on and password-related constraints
- c) Encrypt the data stored in tables
- d) Record the SQL SELECT statements a user executes

### Explanation

Profiles serve two main purposes: limiting database resource use (a) and defining log-on/password constraints (b). Encrypting table data and recording executed statements are not profile functions (the latter is auditing).
