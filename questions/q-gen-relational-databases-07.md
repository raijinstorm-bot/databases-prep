What is a composite key?

- a) A key that is automatically generated by the DBMS
- b) A key composed of more than one column
- c) A key that always allows NULL values
- d) A key that references another table

### Explanation

The presentation defines a composite key as a key composed of more than one column. Because a key must identify each record, the value of each column of such a key must be specified (none can be left empty). A key referencing another table is a foreign key.
