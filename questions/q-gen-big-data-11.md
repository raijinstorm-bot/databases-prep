The "scaling out" answer adopted for Big Data processing is best described as:

- a) buying a single, ever more powerful server
- b) a shared-disk cluster where all servers access one central database
- c) a cluster of commodity hardware hosts, each holding and processing its own part of the data locally, without a relational database
- d) moving all data into a single in-memory relational database

### Explanation

Scaling out uses a cluster of commodity hosts, each storing and processing its own data subset locally and not relying on a relational database (c). Options (a) and (b) describe scaling up and shared-disk approaches that proved insufficient.
