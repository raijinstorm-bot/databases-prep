What exactly does the HBase timestamp stored with every value represent?

- a) The clock time formatted as a human-readable date string
- b) An integer counting milliseconds since 1st January 1970 00:00 until the time of insertion
- c) A sequential version number starting at 1 for each cell
- d) The number of replicas holding the value

### Explanation

The lecture defines the HBase timestamp as an integer measuring the number of milliseconds since 1 January 1970 00:00 up to the time of insertion (b). It is not a formatted date string (a), a sequential counter (c), or a replica count (d).
