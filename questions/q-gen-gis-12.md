What characterizes the enterprise solution for storing geodata?

- a) Geodata is stored in a standard RDBMS such as Oracle or MS SQL Server, or managed by a spatial extension of an RDBMS
- b) Many users can potentially edit the same map layer at the same time
- c) Each layer is restricted to a single editing user, as with files
- d) Descriptive attributes can no longer be linked to geodata

### Explanation

In the enterprise solution geodata lives in a standard RDBMS (or a spatial extension of one), allowing many users to edit the same layer concurrently. The single-user restriction is the file-based drawback, and descriptive data can still be linked.
